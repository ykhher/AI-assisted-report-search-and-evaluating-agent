# Curated Report Discovery Benchmark

A small benchmark for evaluating the full agent pipeline: retrieval, ranking, quality scoring, and lightweight verification. It is designed to be inspectable by hand while providing enough structure to measure each stage objectively.

## Files

| File | Description |
|---|---|
| `queries.csv` | 20 representative report-search queries |
| `documents.csv` | 5 retrieved candidate documents per query (100 rows total) |
| `retrieval_annotations.csv` | Relevance labels and result-class labels per (query, document) pair |
| `quality_annotations.csv` | Document-level DEER-inspired quality labels |

## queries.csv

Each row is one user query with topic, industry, geography, target year range, and intent metadata.

| Field | Description |
|---|---|
| `query_id` | Unique identifier (`q001`–`q020`) |
| `query` | The raw search query text |
| `topic` | Normalised topic label |
| `industry` | Industry sector |
| `geography` | Target geography |
| `target_year_min` / `target_year_max` | Expected report publication year range |
| `intent` | Searcher intent (always `find recent credible industry or research reports`) |

## documents.csv

Each row is one retrieved candidate document. URLs point to real, live reports found via web search.

| Field | Description |
|---|---|
| `doc_id` | Unique identifier (`d001_01` … `d020_05`) |
| `query_id` | Which query this document was retrieved for |
| `retrieved_rank` | Original retrieval rank within the query (1–5) |
| `title` | Document title |
| `source` | Publishing organisation |
| `source_type` | Authority tier: `intergovernmental`, `consulting`, `research_provider`, `vendor`, `news_media`, `unknown`, etc. |
| `url` | Live URL of the document or report page |
| `year` | Publication year |
| `document_type` | One of: `research_report`, `whitepaper`, `landing_page`, `news_article`, `archived_pdf` |
| `is_pdf` | Whether the URL resolves to a PDF |
| `snippet` | Short text description of the document |

### Document type distribution (per query)

| Rank | `document_type` | Intent |
|---|---|---|
| 1 | `research_report` | Ideal result — full report from authoritative source |
| 2 | `whitepaper` | Acceptable — partial methodology, limited transparency |
| 3 | `landing_page` | Weak — topically relevant but no report content |
| 4 | `news_article` | Tangential — shallow mention of the topic |
| 5 | `archived_pdf` | Stale — outdated relative to the query's target years |

## quality_annotations.csv

DEER-inspired quality labels per document, generated by Qwen 2.5-0.5B-Instruct with a structured system prompt and heuristic post-processing for structural fields.

| Field | Scale | Description |
|---|---|---|
| `report_validity_label` | 0–3 | Is this a structured research or industry report? |
| `deer_method_label` | 0–3 | Is a research method, survey, or benchmark process documented? |
| `deer_evidence_label` | 0–3 | Are claims supported by data, references, or statistics? |
| `deer_transparency_label` | 0–3 | Are sources, scope, and limitations clearly stated? |
| `deer_recency_label` | 0–3 | Is the document current for the query's target year range? |
| `authority_label` | 0–3 | How credible is the publishing source? |
| `verification_support_label` | 0–3 | Does the document expose verifiable evidence (PDFs, citations)? |
| `overall_quality_label` | 0–3 | Holistic quality score |
| `rationale` | text | One-sentence justification from Qwen |

## retrieval_annotations.csv

Relevance and ranking labels per (query, document) pair.

| Field | Scale | Description |
|---|---|---|
| `relevance_label` | 0–3 | How relevant is this document to the query? |
| `result_class` | categorical | `good_report`, `mediocre_report`, `weak_page`, `false_positive`, `stale_report` |
| `ranking_preference` | 1–5 | Preference order within the query (1 = best, 5 = worst) |
| `rationale` | text | One-sentence justification |

## Label Scale

All numeric labels use the same 0–3 scale:

| Value | Meaning |
|---|---|
| 3 | Strong / clearly present |
| 2 | Partial / somewhat present |
| 1 | Weak / barely present |
| 0 | Absent / not present |

`ranking_preference` uses 1 (most preferred) to 5 (least preferred) within each query.

## Annotation Method

Annotations are produced by `scripts/annotate_benchmark.py` using a hybrid approach:

- **Qwen 2.5-0.5B-Instruct** scores quality dimensions: `report_validity_label`, `deer_evidence_label`, `deer_transparency_label`, `deer_recency_label`, `authority_label`, `verification_support_label`, `overall_quality_label`, `relevance_label`, and `rationale`.
- **Document-type heuristics** resolve structural fields that the small model cannot reliably classify from short snippets: `result_class`, `ranking_preference`, and `deer_method_label`.

The system prompt instructs Qwen to evaluate each document in the context of its query, apply the DEER framework, and return a single JSON object. Heuristic overrides are applied post-generation.

To re-annotate:

```bash
USE_LOCAL_QWEN=1 /path/to/venv/Scripts/python.exe scripts/annotate_benchmark.py

# Single query only
USE_LOCAL_QWEN=1 /path/to/venv/Scripts/python.exe scripts/annotate_benchmark.py --query q003

# Heuristic-only (no Qwen required)
python scripts/annotate_benchmark.py
```

## URL Maintenance

Document URLs were sourced via web search (SerpApi) for each query. To refresh stale URLs:

```bash
SERPAPI_API_KEY=<key> python scripts/update_document_urls.py
```

The script classifies search results by document type (PDF from authoritative source → `research_report`; vendor page → `landing_page`; etc.) and replaces only rows with placeholder URLs, so partial runs are safe to re-run.

## Intended Uses

- **Retrieval evaluation** — did the agent surface report-like candidates for each query?
- **Ranking evaluation** — did strong reports (`good_report`) rank above weak pages and false positives?
- **Quality scoring calibration** — do scoring coefficients in `source/scoring.py` align with DEER labels?
- **Verification support evaluation** — does the selected top report expose enough evidence for lightweight claim checks?
